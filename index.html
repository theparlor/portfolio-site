Many implements.
One way of making them.

These are composable products — each one a tool you can pick up and use. What binds them isn't a category; it's that every one was forged the same way: noticed, specified, built, and observed through a single operating model. Use the products. Then slip behind the veil to see the machinery.

Hot to cool: molten ore is shaped early, tempered steel ships late. The same heat runs through every implement here.

Behind the veil

None of these came from nowhere.

Every implement above was forged by the same machinery. Two systems guided their development — one is the operating model that built them, the other is the overhead view that holds them together. This is where the warm metalwork cools into the cold architecture underneath.

The operating model

Intent

How each product is actually built — a continuous loop of Notice → Spec → Execute → Observe, replacing ceremony with artifacts that compound. Every product here carries its own .intent/ governance: signals, specs, decisions, traced in git.
